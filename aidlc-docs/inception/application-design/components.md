# 테이블오더 서비스 - 컴포넌트 정의

## 컴포넌트 개요

Next.js App Router 기반 풀스택 애플리케이션으로, 프론트엔드(React)와 백엔드(API Routes)가 하나의 프로젝트에 공존합니다.

---

## 1. 프론트엔드 컴포넌트

### 1.1 Customer UI (고객용 인터페이스)
- **목적**: 테이블 태블릿에서 고객이 메뉴 조회, 장바구니 관리, 주문을 수행하는 화면
- **책임**:
  - 테이블 자동 로그인 및 세션 관리
  - 카테고리별 메뉴 조회 및 탐색
  - 장바구니 관리 (로컬 저장)
  - 주문 생성 및 확인
  - 주문 내역 조회 및 SSE 기반 실시간 상태 업데이트
- **관련 스토리**: US-C01, US-C02, US-C03, US-C04, US-C05

### 1.2 Admin UI (관리자용 인터페이스)
- **목적**: 매장 관리자가 주문 모니터링, 테이블 관리, 메뉴 관리를 수행하는 화면
- **책임**:
  - 실시간 주문 대시보드 (SSE 기반)
  - 주문 상태 변경
  - 테이블 초기 설정 및 세션 관리
  - 주문 삭제, 매장 이용 완료 처리
  - 과거 주문 내역 조회
  - 메뉴 CRUD 및 이미지 업로드
- **관련 스토리**: US-A01 ~ US-A09

---

## 2. 백엔드 컴포넌트

### 2.1 Auth Module (인증 모듈)
- **목적**: 테이블 태블릿 인증 및 세션 토큰 관리
- **책임**:
  - 테이블 로그인 (매장코드 + 테이블번호)
  - 세션 토큰 발급 및 검증
  - 테이블 세션 라이프사이클 관리

### 2.2 Menu Module (메뉴 모듈)
- **목적**: 메뉴 및 카테고리 데이터 관리
- **책임**:
  - 카테고리 CRUD
  - 메뉴 항목 CRUD
  - 메뉴 이미지 업로드/저장 (로컬 파일 시스템)
  - 메뉴 노출 순서 관리
  - 데이터 검증 (필수 필드, 가격 범위)

### 2.3 Order Module (주문 모듈)
- **목적**: 주문 생성, 상태 관리, 조회
- **책임**:
  - 주문 생성 (주문번호 자동 생성)
  - 주문 상태 전이 (pending → preparing → completed)
  - 테이블 세션별 주문 조회
  - 주문 삭제 (관리자)
  - 주문 금액 계산

### 2.4 Table Module (테이블 모듈)
- **목적**: 테이블 및 세션 관리
- **책임**:
  - 테이블 등록 및 설정
  - 테이블 세션 시작/종료
  - 매장 이용 완료 처리 (세션 종료 + 주문 이력 이동)
  - 과거 주문 내역 조회

### 2.5 SSE Module (실시간 통신 모듈)
- **목적**: Server-Sent Events 기반 실시간 데이터 전송
- **책임**:
  - SSE 연결 관리 (클라이언트 연결/해제)
  - 주문 이벤트 브로드캐스트 (신규 주문, 상태 변경, 주문 삭제)
  - 테이블별/관리자별 이벤트 라우팅

### 2.6 Database Module (데이터 접근 모듈)
- **목적**: SQLite 데이터베이스 접근 및 관리
- **책임**:
  - 데이터베이스 연결 관리
  - 스키마 마이그레이션
  - 시드 데이터 관리
  - 각 엔티티별 데이터 접근 함수 제공

### 2.7 Upload Module (파일 업로드 모듈)
- **목적**: 메뉴 이미지 파일 업로드 및 서빙
- **책임**:
  - 이미지 파일 수신 및 저장 (로컬 파일 시스템)
  - 파일 경로 관리
  - 정적 파일 서빙
