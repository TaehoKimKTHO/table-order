# 테이블오더 서비스 - 사용자 스토리

## Epic 1: 고객 주문 경험

### US-C01: 테이블 자동 로그인
**As a** 고객 (김고객)
**I want to** 테이블에 앉으면 별도 로그인 없이 바로 주문 화면을 볼 수 있기를
**So that** 번거로운 인증 절차 없이 즉시 주문할 수 있다

**Acceptance Criteria:**
- [ ] 관리자가 초기 설정한 테이블 정보(매장 식별자, 테이블 번호)가 로컬에 저장된다
- [ ] 저장된 정보로 자동 로그인이 수행된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 표시된다
- [ ] 세션은 관리자가 이용 완료 처리할 때까지 유지된다

---

### US-C02: 메뉴 조회 및 탐색
**As a** 고객 (김고객)
**I want to** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를
**So that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다

**Acceptance Criteria:**
- [ ] 메뉴 화면이 기본 화면으로 표시된다
- [ ] 카테고리별로 메뉴가 분류되어 표시된다
- [ ] 각 메뉴에 메뉴명, 가격, 설명, 이미지가 표시된다
- [ ] 카테고리 간 빠른 이동이 가능하다
- [ ] 카드 형태 레이아웃으로 표시된다
- [ ] 터치 타겟이 최소 44x44px 이상이다

---

### US-C03: 장바구니 관리
**As a** 고객 (김고객)
**I want to** 주문할 메뉴를 장바구니에 담고 수량을 조절하기를
**So that** 주문 전에 선택한 메뉴를 확인하고 수정할 수 있다

**Acceptance Criteria:**
- [ ] 메뉴를 장바구니에 추가할 수 있다
- [ ] 장바구니에서 메뉴를 삭제할 수 있다
- [ ] 수량을 증가/감소할 수 있다
- [ ] 총 금액이 실시간으로 계산되어 표시된다
- [ ] 장바구니를 비울 수 있다
- [ ] 페이지 새로고침 시에도 장바구니가 유지된다 (로컬 저장)

---

### US-C04: 주문 생성
**As a** 고객 (김고객)
**I want to** 장바구니의 메뉴를 주문으로 확정하기를
**So that** 매장에 주문이 전달되어 음식을 받을 수 있다

**Acceptance Criteria:**
- [ ] 주문 내역을 최종 확인할 수 있다
- [ ] 주문 확정 버튼으로 주문을 생성할 수 있다
- [ ] 주문 성공 시 주문 번호가 표시된다
- [ ] 주문 성공 시 장바구니가 자동으로 비워진다
- [ ] 주문 성공 후 5초 뒤 메뉴 화면으로 자동 리다이렉트된다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다

---

### US-C05: 주문 내역 조회
**As a** 고객 (김고객)
**I want to** 현재 테이블에서 주문한 내역과 상태를 확인하기를
**So that** 주문이 정상 접수되었는지, 현재 진행 상태가 어떤지 알 수 있다

**Acceptance Criteria:**
- [ ] 주문이 시간 순으로 정렬되어 표시된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다
- [ ] 주문 상태(대기중/준비중/완료)가 SSE로 실시간 업데이트된다
- [ ] 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)

---

## Epic 2: 관리자 주문 관리

### US-A01: 실시간 주문 모니터링
**As a** 매장 관리자 (이사장)
**I want to** 들어오는 주문을 실시간으로 대시보드에서 확인하기를
**So that** 주문을 놓치지 않고 빠르게 처리할 수 있다

**Acceptance Criteria:**
- [ ] SSE 기반으로 주문 목록이 실시간 업데이트된다
- [ ] 테이블별 카드 형태의 그리드 레이아웃으로 표시된다
- [ ] 각 테이블 카드에 총 주문액이 표시된다
- [ ] 주문 카드 클릭 시 전체 메뉴 목록 상세를 볼 수 있다
- [ ] 신규 주문이 2초 이내에 표시된다
- [ ] 신규 주문이 시각적으로 강조된다 (색상 변경, 애니메이션)

---

### US-A02: 주문 상태 변경
**As a** 매장 관리자 (이사장)
**I want to** 주문 상태를 대기중에서 준비중, 완료로 변경하기를
**So that** 고객에게 주문 진행 상황을 알려줄 수 있다

**Acceptance Criteria:**
- [ ] 주문 상태를 대기중 → 준비중 → 완료로 변경할 수 있다
- [ ] 상태 변경이 고객 화면에 SSE로 실시간 반영된다
- [ ] 테이블별 필터링으로 특정 테이블 주문만 볼 수 있다

---

## Epic 3: 관리자 테이블 관리

### US-A03: 테이블 초기 설정
**As a** 매장 관리자 (이사장)
**I want to** 테이블 번호와 비밀번호를 설정하기를
**So that** 고객 태블릿이 자동 로그인할 수 있도록 준비할 수 있다

**Acceptance Criteria:**
- [ ] 테이블 번호를 설정할 수 있다
- [ ] 설정 완료 후 해당 테이블의 자동 로그인이 활성화된다

---

### US-A04: 주문 삭제
**As a** 매장 관리자 (이사장)
**I want to** 잘못된 주문을 삭제하기를
**So that** 오류 주문을 정정하고 정확한 주문 현황을 유지할 수 있다

**Acceptance Criteria:**
- [ ] 특정 주문의 삭제 버튼을 클릭할 수 있다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 확인 시 주문이 즉시 삭제된다
- [ ] 삭제 후 테이블 총 주문액이 재계산된다
- [ ] 성공/실패 피드백이 표시된다

---

### US-A05: 매장 이용 완료 처리
**As a** 매장 관리자 (이사장)
**I want to** 고객이 퇴장할 때 테이블 이용 완료를 처리하기를
**So that** 다음 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다

**Acceptance Criteria:**
- [ ] 이용 완료 버튼 클릭 시 확인 팝업이 표시된다
- [ ] 확인 시 테이블 세션이 종료된다
- [ ] 해당 세션의 주문 내역이 과거 이력으로 이동된다
- [ ] 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다
- [ ] 성공/실패 피드백이 표시된다

---

### US-A06: 과거 주문 내역 조회
**As a** 매장 관리자 (이사장)
**I want to** 테이블별 과거 주문 내역을 조회하기를
**So that** 이전 매출과 주문 이력을 확인할 수 있다

**Acceptance Criteria:**
- [ ] 과거 내역 버튼으로 조회 화면에 진입할 수 있다
- [ ] 테이블별 과거 주문 목록이 시간 역순으로 표시된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
- [ ] 날짜 필터링이 가능하다
- [ ] 닫기 버튼으로 대시보드로 복귀할 수 있다

---

## Epic 4: 관리자 메뉴 관리

### US-A07: 메뉴 조회
**As a** 매장 관리자 (이사장)
**I want to** 카테고리별로 등록된 메뉴를 조회하기를
**So that** 현재 메뉴 현황을 파악할 수 있다

**Acceptance Criteria:**
- [ ] 카테고리별로 메뉴 목록이 표시된다
- [ ] 각 메뉴의 이름, 가격, 설명, 이미지, 판매 상태가 표시된다

---

### US-A08: 메뉴 등록
**As a** 매장 관리자 (이사장)
**I want to** 새로운 메뉴를 등록하기를
**So that** 고객에게 새 메뉴를 제공할 수 있다

**Acceptance Criteria:**
- [ ] 메뉴명, 가격, 설명, 카테고리를 입력할 수 있다
- [ ] 메뉴 이미지를 업로드할 수 있다
- [ ] 필수 필드(메뉴명, 가격, 카테고리) 검증이 수행된다
- [ ] 가격 범위 검증이 수행된다
- [ ] 등록 성공/실패 피드백이 표시된다

---

### US-A09: 메뉴 수정 및 삭제
**As a** 매장 관리자 (이사장)
**I want to** 기존 메뉴를 수정하거나 삭제하기를
**So that** 메뉴 정보를 최신 상태로 유지할 수 있다

**Acceptance Criteria:**
- [ ] 메뉴 정보(이름, 가격, 설명, 카테고리, 이미지)를 수정할 수 있다
- [ ] 메뉴를 삭제할 수 있다
- [ ] 메뉴 노출 순서를 조정할 수 있다
- [ ] 수정/삭제 성공/실패 피드백이 표시된다
