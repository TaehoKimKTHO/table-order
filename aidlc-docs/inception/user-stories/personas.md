# 테이블오더 서비스 - 사용자 페르소나

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|---|---|
| **이름** | 김고객 |
| **역할** | 매장 방문 고객 |
| **설명** | 매장 테이블에 앉아 태블릿을 통해 메뉴를 조회하고 주문하는 사용자 |
| **목표** | 빠르고 편리하게 메뉴를 탐색하고 주문하기 |
| **기술 수준** | 다양 (스마트폰/태블릿 기본 사용 가능) |
| **사용 환경** | 매장 내 테이블에 설치된 태블릿 (브라우저 기반) |
| **핵심 니즈** | 직관적인 UI, 빠른 주문 프로세스, 주문 상태 확인 |
| **불편 사항** | 복잡한 로그인 절차, 느린 응답, 이전 손님 주문 내역 노출 |

### 주요 사용 시나리오
1. 테이블에 앉아 태블릿으로 메뉴 탐색
2. 원하는 메뉴를 장바구니에 담고 수량 조절
3. 주문 확정 후 주문 번호 확인
4. 추가 주문 시 동일 프로세스 반복
5. 주문 내역에서 현재 주문 상태 확인

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 이사장 |
| **역할** | 매장 운영자/관리자 |
| **설명** | 매장의 주문을 실시간으로 모니터링하고, 테이블과 메뉴를 관리하는 사용자 |
| **목표** | 효율적인 매장 운영, 실시간 주문 파악, 메뉴/테이블 관리 |
| **기술 수준** | 중간 (PC/태블릿 기본 사용 가능) |
| **사용 환경** | 매장 카운터 PC 또는 태블릿 (브라우저 기반) |
| **핵심 니즈** | 실시간 주문 알림, 간편한 주문 상태 변경, 테이블 세션 관리 |
| **불편 사항** | 주문 누락, 복잡한 관리 화면, 느린 주문 반영 |

### 주요 사용 시나리오
1. 대시보드에서 실시간 주문 모니터링
2. 신규 주문 확인 후 상태 변경 (대기중 → 준비중 → 완료)
3. 테이블 초기 설정 (번호, 비밀번호)
4. 고객 퇴장 시 테이블 이용 완료 처리
5. 메뉴 등록/수정/삭제 관리
6. 잘못된 주문 삭제 처리
7. 과거 주문 내역 조회
